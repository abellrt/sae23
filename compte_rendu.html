Gestion de projet

Introduction :

Nous avons réalisé ce projet dans le cadre de la SAé 23 “Mettre en place une solution informatique pour l’entreprise”, présentée au semestre 2. Il nous a été confié par nos enseignants dans un contexte professionnel simulé.
L’objectif était d’exploiter les données de capteurs IoT répartis dans les bâtiments de l'IUT et de les rendre accessibles via une interface web.

Pendant plusieurs semaines, nous avons travaillé en groupe pour concevoir et déployer une solution complète, en mobilisant les compétences acquises en bases de données et en développement web, dans l’optique de répondre aux attentes de la problématique.

Problématique

Comment concevoir et mettre en place une solution informatique complète permettant la collecte, le stockage et la visualisation de données issues de capteurs IoT, tout en offrant une interface web sécurisée, adaptée aux différents profils d'utilisateurs de l'IUT ?

Les solutions proposées :

Pour répondre à la problématique, deux solutions ont été demandées. La première repose sur une chaîne de conteneurs Docker associant Mosquitto, Node-RED, InfluxDB et Grafana. Elle présente l'avantage d'un prototypage rapide avec peu de code à produire et offre une interface claire pour le gestionnaire, mais possède d’importantes limites : absence de gestion de comptes, accès Node-RED non sécurisé et peu de flexibilité pour des affichages personnalisés.
La seconde solution est un site web dynamique développé avec PHP et MySQL. Plus exigeante en termes de développement, elle permet en contrepartie une gestion fine des utilisateurs et des accès, une sécurisation des pages via des sessions, ainsi que des formulaires adaptés aux besoins de chaque profil.

Docker (Labourse--Sarnin):
La chaîne de traitement mise en place permet de visualiser les données des capteurs déployés dans les bâtiments de l'IUT. Elle repose sur quatre outils, chacun hébergé dans un conteneur Docker dédié : Mosquitto, Node-RED, InfluxDB et Grafana.
Les capteurs publient leurs mesures via MQTT, Mosquitto les redistribue, Node-RED les récupère et les convertit, InfluxDB les enregistre dans la base de données et Grafana les affiche sous forme de dashboard.


Figure 3 : dockers présents sur la VM
1. Mosquitto
Mosquitto joue le rôle de récupérateur et de distributeur des messages MQTT. Les capteurs publient leurs données sur le broker MQTT de l’IUT, et il les redistribue à tous mes abonnés comme Node-RED.
2 Node-RED
Node-RED est le cœur de la chaîne. Il souscrit aux topics MQTT des capteurs, convertit les données reçues en valeurs numériques, puis les insère dans la base InfluxDB.


Figure 4 : flow Node-RED
Le flow est composé de trois nœuds enchaînés :
  • mqtt in : souscription aux topics des capteurs
  • change : conversion du payload en nombre ($number(payload))
  • influxdb out : écriture dans la base capteurs
En parallèle, un dashboard Node-RED affiche, en temps réel, une jauge par capteur, regroupées par bâtiment.


Figure 5 : dashboard Node-RED
3 InfluxDB InfluxDB stocke chaque mesure reçue avec un horodatage précis, le nom de la salle et celui du bâtiment en tant que « tags », et la valeur du capteur en tant que « field ».


Figure 6 : base de données influfDB
4 Grafana
Grafana se connecte à InfluxDB comme source de données et permet de construire un dashboard présentant l'ensemble des capteurs des deux bâtiments. Chaque panel affiche l'évolution de la mesure d'une salle, avec un rafraîchissement automatique toutes les 30 secondes.


Figure 7 : exemple dashboard d’une salle de cours


Figure 8 : paramétrage d’un des graphique d’une des salles

Ardouin: Scripts


Afin d’alimenter la base de données du site web avec les mesures issues des capteurs IoT, nous avons développé un script en PHP exécuté en ligne de commande sur la machine virtuelle Linux. Son rôle est de récupérer automatiquement les données publiées sur le broker MQTT, de les décoder, puis de les enregistrer dans la base MySQL utilisée par notre application web.
Fonctionnement général du script
Le script commence par se connecter au broker MQTT de l’IUT à l’aide de la commande mosquitto_sub. Cette commande permet d’écouter un topic précis correspondant à une salle équipée de capteurs, ici par exemple :

Figure 10 : exemple de résultat de la commande mosquitto_sub
Le script récupère un message au format JSON contenant les différentes mesures envoyées par le capteur, telles que :
  • la température ;
  • le taux de CO₂ ;
  • l’humidité ;
  • la pression atmosphérique.

Une fois le message reçu, il est décodé grâce à la fonction json_decode(), ce qui permet d’accéder individuellement aux différentes valeurs mesurées ainsi qu’au nom du capteur.
Insertion des données dans la base : Après la récupération des données, le script établit une connexion à la base de données MySQL. Les informations extraites du message sont ensuite insérées dans la table mesures. Pour chaque message reçu, plusieurs enregistrements sont ajoutés : un pour la température, un pour le CO₂, un pour l’humidité et un pour la pression. Chaque insertion contient :
  • le nom du capteur ;
  • la date de la mesure ;
  • l’heure de la mesure ;
  • la valeur mesurée.
Le nom du capteur est construit de manière à identifier à la fois le type de mesure et le capteur concerné, par exemple temperature_AM107-33 ou co2_AM107-4.


Figure 11 : donné enregistrée dans la table de donnée
Intérêt du script dans le projet :
Ce script permet le lien entre les capteurs IoT et la base de données du site. Grâce à lui, les données sont récupérées automatiquement sans intervention manuelle, puis stockées de façon exploitable pour l’affichage sur les pages web.
Il permet donc :
d’automatiser la collecte des mesures, de conserver un historique des données reçues, d’alimenter les pages de consultation avec des valeurs récentes, de préparer les données pour d’éventuels traitements ou graphiques.
Difficultés rencontrées :
La mise en place de ce script a demandé plusieurs ajustements. En effet, le format des données reçues a dû être vérifié et adapté au traitement PHP, et la structure de la base de données a évolué au fil du projet. De plus, l’automatisation du lancement du script a nécessité l’utilisation d’une tâche planifiée (crontab) afin d’exécuter régulièrement la récupération des données.

Conclusion

Ce projet nous a permis de mettre en pratique, dans un cadre concret et professionnalisant, les compétences acquises au cours du semestre en bases de données, développement web, administration système et gestion de projet. À travers la réalisation des deux solutions demandées, nous avons pu découvrir deux approches différentes de la valorisation de données issues de capteurs IoT.

La première solution, basée sur une architecture Docker avec Mosquitto, Node-RED, InfluxDB et Grafana, nous a permis de mettre en place rapidement une chaîne fonctionnelle de collecte, de traitement et de visualisation des données. Elle constitue une solution efficace pour le prototypage et l’affichage technique, mais reste limitée en matière de sécurité, de personnalisation et de gestion des utilisateurs.

La seconde solution, fondée sur un site web dynamique en PHP/MySQL, a demandé davantage de travail de conception et de développement, mais elle répond plus précisément au cahier des charges. Elle offre une meilleure gestion des comptes, des droits d’accès et de l’affichage des données selon les profils utilisateurs, tout en assurant une interface plus adaptée à une utilisation réelle au sein de l’IUT.

Malgré certaines difficultés rencontrées au cours du projet, notamment lors de l’automatisation des scripts, de la mise en place de la base de données ou encore du démarrage des conteneurs, nous avons su nous organiser, nous répartir les tâches et adapter notre planning.

Le cahier des charges a été respecté à 75 %, car tout d'abord le script de récupération de données de ne s'adapter pas s'il y a l'ajout d'un nouveau capteur. Ensuite le faites qu'il n'y a pas de liste déroulant pour choisir le capteur, et même choisir l'horaire ou l'on veut voir les données, c'est pour cella qu'on ne remplit pas entièrement le cahier des charges.
